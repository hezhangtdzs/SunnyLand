# 实现最高分储存功能

## 功能需求
- 添加两个关卡的最高分储存
- 每次分数大于当前关卡最高分就记录
- 回到第一关时清空当前分数

## 实现步骤

### 1. 修改 SessionData 头文件 (`src/game/data/session_data.h`)
- 添加两个成员变量：`int high_score_level1_` 和 `int high_score_level2_`
- 添加对应的 getter 方法：`getHighScoreLevel1()` 和 `getHighScoreLevel2()`
- 添加更新最高分的方法：`updateHighScore()`
- 添加检查是否回到第一关并清空分数的方法：`checkAndResetScore()`

### 2. 修改 SessionData 实现文件 (`src/game/data/session_data.cpp`)
- 在构造函数中初始化最高分变量为 0
- 实现 `updateHighScore()` 方法，根据当前关卡更新对应的最高分
- 实现 `checkAndResetScore()` 方法，检查是否回到第一关并清空当前分数
- 修改 `toJson()` 方法，添加最高分到 JSON 序列化
- 修改 `fromJson()` 方法，从 JSON 反序列化最高分
- 修改 `reset()` 方法，保持最高分不变

### 3. 集成到游戏逻辑
- 在添加分数时调用 `updateHighScore()` 方法
- 在切换关卡时调用 `checkAndResetScore()` 方法

## 技术细节
- 使用现有的 JSON 序列化/反序列化机制保存最高分
- 保持与现有代码风格和架构的一致性
- 添加适当的日志记录，便于调试
- 确保最高分在游戏重启后仍然保留
